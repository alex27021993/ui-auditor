# Changelog

Все заметные изменения методики и инструмента **UI Auditor**.
Версии: **MINOR** (1.X.0) — новые проверки/возможности, **PATCH** (1.0.X) — исправления.
Текущая версия — в файле [`VERSION`](VERSION); `install.sh` печатает её при запуске.
Формат даты — ГГГГ-ММ-ДД.

## [1.5.1] — 2026-06-22
### Добавлено
- Команды `/audit` и `/audit-visual` печатают версию методики (`UI Auditor v<версия>`)
  в начале отчёта, читая её динамически из файла `VERSION` (без правки команды при
  каждом релизе).

## [1.5.0] — 2026-06-22
### Добавлено
- **Зона 7b «Автоудаление out-of-stock товара из корзины»** — клиент не должен дойти
  до оплаты с OOS-товаром. Проверки: барьер на добавление OOS; валидация стока при
  открытии корзины (есть ли AJAX-эндпоинт автоочистки, не удаляется ли in-stock);
  реальное удаление при переводе товара в OOS (если есть доступ в админку); барьер на
  чекауте. С градацией severity и оговоркой об ограничении без доступа в админку.

## [1.4.0] — 2026-06-18
### Добавлено
- Слэш-команды `/audit <url>` и `/audit-visual <url>` — запуск аудита по одному URL,
  без копирования текста промпта. Команды ссылаются на скил, поэтому всегда применяют
  актуальную методику из `SKILL.md`.
- `install.sh` теперь ставит и команды в `~/.claude/commands/`.
### Исправлено
- `.gitignore` больше не прячет `commands/audit-visual.md` (правило `audit-*.md` для
  отчётов задевало файл команды).

## [1.3.0] — 2026-06-18
### Добавлено
- **Зона 7a «Живой DOM мини-корзины после AJAX»** — класс скрытых багов, когда после
  AJAX-добавления дровер подменяется фрагментами, а обработчики не перевешиваются и
  второе действие в корзине молча не работает. Проверки: +/− через Network, повтор-
  триггер, удалить→добавить (белый дровер), купон (нативный Enter, сообщение, двойная
  отправка), три счётчика после каждого шага.

## [1.2.0] — 2026-06-15
### Добавлено
- **Предохранители от ложных находок** (дисциплина вердикта): достижимость через UI
  прежде вердикта о баге URL-уровня; синтетический клик ≠ реальный ввод; фильтр
  «без эффекта» — сначала проверить редкое значение.
- Проверка корзины (A2): три счётчика после AJAX-add — дровер / бейдж хедера / после F5.
- Проверка PDP (A4): цена вариации обновляется и совпадает с ценой в корзине.
- Проверка промо (A5): вменяемость даты доставки («Estimated delivery» — в будущем,
  считается относительно сегодня, а не захардкожена).

## [1.1.0] — 2026-06-15
### Добавлено
- Установка для коллег через `install.sh` (симлинки скилов в `~/.claude/skills/`).
- Бандл браузера: `.mcp.json` с Playwright MCP + раздел «Браузер» в README.
### Изменено
- Скилы стали tool-agnostic — описание инструментов браузера по возможности, а не
  по жёстким именам (подходит любой браузерный MCP).

## [1.0.0] — 2026-06-15
### Добавлено
- Первый релиз. Два скила: `ecommerce-functional-audit` (Трек A — «работает ли и
  логично ли») и `ecommerce-visual-audit` (Трек B — «как выглядит и корректна ли
  информация»). Методика и формат отчёта — в `CLAUDE.md` и `SKILL.md`.

---

### Как добавлять запись (при каждом изменении методики)
1. Подними версию в [`VERSION`](VERSION): новая проверка/возможность → MINOR (1.X.0),
   исправление → PATCH (1.0.X).
2. Добавь блок `## [версия] — ГГГГ-ММ-ДД` сверху с разделами «Добавлено / Изменено /
   Исправлено».
3. Закоммить вместе с правкой `SKILL.md` в том же PR.
